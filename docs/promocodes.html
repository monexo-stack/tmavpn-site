Промокоды и рефералы

Два механизма привлечения и удержания: промокоды со скидками и реферальная программа с двумя режимами вознаграждения.

Промокоды

Промокоды дают скидку на покупку подписки. Поддерживаются два типа скидок: процентная и фиксированная.

Типы скидок

ТипПолеПримерРасчёт
Процентная percent (1–100) Промокод SALE20 с 20% Скидка = цена × 20 / 100
Фиксированная amount (в рублях) Промокод GIFT50 на 50 ₽ Скидка = min(50, цена)

Параметры промокода

ПолеОписание
textТекст промокода (автоматически переводится в верхний регистр)
discount_typepercentage или fixed
percentПроцент скидки (для типа percentage)
amountФиксированная сумма скидки в рублях (для типа fixed)
max_usesМаксимальное количество использований (null = без ограничений)
expires_atДата истечения (null = бессрочный)
activeВключён/выключен

Ограничения

  • Один пользователь может использовать один промокод только один раз
  • Промокод проверяется при создании платежа — если невалиден, платёж создаётся без скидки
  • При использовании в платеже сохраняется информация о промокоде и размере скидки

Статусы промокода

СтатусУсловие
activeВключён, не истёк, лимит не достигнут
expiredИстёк срок действия или исчерпан лимит использований
disabledОтключён администратором

Управление в админке

Раздел Promocodes в админ-панели:

  • Создание — текст, тип скидки, значение, лимит, дата истечения
  • Включение/отключение — через переключатель в таблице
  • Удаление — каскадно удаляет записи использования, обнуляет ссылку в платежах
  • Статистика — количество использований, общая сумма скидок

Реферальная система

Каждый пользователь может приглашать других через реферальную ссылку. Система поддерживает два режима вознаграждения.

Режимы реферальной программы

Бонусные дни (days)
Партнёр получает дополнительные дни подписки за каждого приглашённого. Режим по умолчанию. Дни автоматически добавляются к подписке партнёра.
Денежное вознаграждение (money)
Партнёр получает процент от каждой оплаты приглашённого пользователя. Накопленный баланс можно вывести через FreeKassa.

Как работают бонусные дни

Бонус за приглашение (invite) — начисляется однократно, когда приглашённый активирует пробный период и использует минимум 3 МБ трафика.

Бонус за покупку (purchase) — начисляется при каждой оплате приглашённым пользователем. Количество дней зависит от тарифа:

Тариф приглашённогоДней партнёру
1 месяц5
3 месяца10
6 месяцев15
1 год20
Автоматическое применение

Бонусные дни автоматически добавляются к подписке партнёра, если она активна. Если подписка неактивна — дни сохраняются как «ожидающие» и могут быть применены позже.

Жизненный цикл реферала

Пользователь делится реферальной ссылкой
В Mini App есть кнопка «Пригласить друга», которая генерирует ссылку вида t.me/bot?start=ref_XXXXX.
Приглашённый регистрируется
При регистрации в поле referral_user_tgid сохраняется ID пригласившего. Связь устанавливается навсегда.
Приглашённый активирует trial
После активации пробного периода и использования 3+ МБ трафика — партнёру начисляется бонус за приглашение (однократно).
Приглашённый оплачивает подписку
При каждой оплате — партнёру начисляется бонус за покупку (дни или деньги, в зависимости от режима). Бонусы начисляются при каждой покупке без ограничений.

Защита от дублирования

  • Бонус за приглашение — максимум один на пару «партнёр + приглашённый»
  • Бонус за покупку — максимум один на платёж (система предотвращает повторное начисление)
  • Требование трафика (3 МБ) для invite-бонуса предотвращает злоупотребление ботами

Статистика в админке

Раздел Referrals показывает:

  • Список всех партнёров с количеством приглашённых
  • Общее количество начисленных бонусных дней (invite + purchase отдельно)
  • Конверсия: процент партнёров, чьи рефералы совершили покупку
  • Топ-5 партнёров по количеству приглашённых
  • Топ-5 партнёров по начисленным дням
  • Детальный список приглашённых для каждого партнёра

Выплаты

В режиме money партнёры накапливают баланс в рублях и могут запросить вывод средств.

Процесс вывода

Партнёр запрашивает вывод
В Mini App: указывает сумму и реквизиты (номер карты или кошелька). Минимальная сумма — 500 ₽ (настраивается). Аккаунт должен быть старше 30 дней. Лимит — 1 запрос в 24 часа.
Система блокирует баланс
Сумма атомарно списывается с баланса партнёра. Создаётся заявка со статусом pending.
Админ одобряет или отклоняет
При одобрении — система отправляет запрос в FreeKassa API для перевода средств. При отклонении — сумма возвращается на баланс партнёра.
FreeKassa обрабатывает выплату
Статус обновляется на completed или failed. Пользователь получает уведомление через Telegram-бота.

Статусы заявок на вывод

СтатусОписание
pendingОжидает рассмотрения администратором
processingОдобрена, отправлена в FreeKassa
completedДеньги переведены
failedОшибка на стороне FreeKassa
rejectedОтклонена администратором, баланс возвращён

Настройки реферальной системы

В админ-панели можно настроить следующие параметры реферальной программы:

  • Режим вознаграждения — бонусные дни или денежное вознаграждение (по умолчанию — дни)
  • Количество дней за приглашение — сколько дней получает партнёр за каждого нового реферала (по умолчанию — 3)
  • Количество дней за покупку — сколько дней начисляется за покупку приглашённого (настраивается по тарифам)
  • Процент комиссии — какой процент от оплаты получает партнёр в денежном режиме (по умолчанию — 10%)
  • Минимальная сумма вывода — минимальная сумма для запроса выплаты (по умолчанию — 500 ₽)
