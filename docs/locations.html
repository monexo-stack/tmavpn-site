Локации и серверы

Управление VPN-инфраструктурой: локации (страны), серверы, мониторинг, балансировка нагрузки, провижининг ключей.

Концепция

Инфраструктура организована в два уровня:

  • Локация — географическая точка (страна/город). Например: Нидерланды, Германия, США.
  • Сервер — конкретный VPS внутри локации с установленной 3x-ui панелью. В одной локации может быть несколько серверов для распределения нагрузки.

При покупке подписки пользователь получает ключи на всех локациях. В каждой локации ключ создаётся на одном из серверов — на том, где меньше всего пользователей (автоматическая балансировка).

Локации

Данные локации

ПолеОписание
nameНазвание локации (отображается в Mini App и VPN-клиенте)
countryСтрана
country_codeISO код страны (NL, DE, US)
flagФлаг страны
statusonline / offline / maintenance
traffic_limit_gbЛимит трафика на пользователя (NULL = безлимит)

Действия с локацией

  • Редактировать — изменить название, лимит трафика, статус. При изменении названия система автоматически обновляет его во всех VPN-конфигурациях пользователей.
  • Добавить сервер — открывает форму добавления сервера в эту локацию.
  • Провижининг ключей — вручную создать ключи для всех пользователей с подпиской, у которых ещё нет ключа в этой локации.
  • Баланс нагрузки — просмотреть распределение пользователей между серверами.
  • Ребалансировка — автоматически перераспределить пользователей между серверами.
  • Удалить — удаляет локацию со всеми серверами. Опционально удаляет ключи пользователей.

Серверы

Данные сервера

ПолеОписание
nameИмя сервера (авто: NL-1, DE-2). Генерируется как КОД_СТРАНЫ-НОМЕР
ipIP-адрес сервера (уникальный)
statusonline / offline / degraded / maintenance
max_spaceМаксимальная вместимость (рекомендуется по результатам speedtest)
current_usersТекущее количество активных пользователей
bandwidthСкорость канала (из speedtest)
x3ui_urlURL панели 3x-ui
x3ui_connectedСтатус подключения к панели

Статусы серверов

СтатусОписание
onlineСервер работает, принимает подключения
degradedНестабильная работа (прерывистая связь с агентом)
offlineСервер не отвечает более 5 минут
maintenanceРежим обслуживания (установлен администратором)

Действия с сервером

  • Редактировать — изменить параметры, учётные данные 3x-ui, SSH
  • Проверка соединения — тест SSH + 3x-ui + обновление статуса
  • Speedtest — замер скорости канала. По результатам рассчитывается рекомендуемая вместимость
  • Перезапуск Xray — перезапуск VPN-сервиса на сервере через 3x-ui
  • Установить агент — установка мониторинг-агента по SSH
  • Обслуживание — переключить в режим maintenance (новые ключи не создаются)
  • Удалить — удаляет сервер, отключает агент, удаляет клиентов из 3x-ui

Пример: Вы добавили новый сервер в локацию «Нидерланды», где уже есть 500 пользователей. После добавления нажмите «Провижининг ключей» — система автоматически создаст ключи для всех 500 пользователей на новом сервере. Прогресс отображается в реальном времени.

VPN Agent

На каждый VPN-сервер устанавливается агент — скрипт, который передаёт метрики в реальном времени через WebSocket.

Что агент отправляет

  • Загрузка CPU (%)
  • Использование RAM (GB)
  • Использование диска (GB)
  • Сетевой трафик (upload / download)
  • Список активных VPN-подключений (IP, email, трафик каждого)
  • Статус компонентов (xray-core, fail2ban)

Установка агента

Агент устанавливается автоматически при добавлении сервера (если прошла полная проверка) или вручную через кнопку «Установить агент». Требуется SSH-доступ к серверу.

Статусы агента

СтатусОписание
onlineПодключён, передаёт метрики
offlineНе отвечает более 5 минут
degradedНестабильное соединение
not_installedАгент не установлен

Агент использует гибридный режим: основное соединение через WebSocket, резервное — HTTP-опрос каждые 10 секунд. При потере WebSocket-соединения автоматически переключается на HTTP и обратно.

Мониторинг

В верхней части страницы локаций отображаются 4 карточки мониторинга:

Серверы
Количество онлайн серверов из общего числа. Зелёный — все работают. Жёлтый — часть офлайн. Красный — все офлайн.
Агенты
Количество подключённых агентов из установленных. Показывает состояние мониторинга.
Проблемы
Количество проблемных серверов: офлайн, degraded или агент офлайн при работающем сервере. При наведении — список проблем.
Доступные подписки
Минимальное количество свободных мест по всем локациям. Показывает «узкое горлышко» — локацию с наименьшей ёмкостью.

Метрики в реальном времени

Для каждого сервера с установленным агентом отображаются:

  • CPU — текущая загрузка процессора
  • RAM — использование оперативной памяти
  • Диск — использование диска
  • Трафик — upload / download в GB
  • Подключения — количество активных VPN-соединений

Данные обновляются каждые 5 секунд через WebSocket.

Провижининг ключей

Провижининг — автоматическое создание VPN-ключей для пользователей на серверах в локации.

Когда происходит

  • Автоматически — при добавлении нового сервера в локацию
  • Вручную — через меню локации → «Провижининг ключей»
  • Фоновая задача — каждые 2 минуты проверяет и создаёт отсутствующие ключи

Как работает

Поиск пользователей
Система находит пользователей с активной подпиской, у которых нет ключа в данной локации.
Выбор сервера
Для каждого пользователя выбирается наименее загруженный сервер в локации с свободными местами.
Создание ключа
На 3x-ui панели создаётся клиент с VLESS-конфигурацией. Ключ сохраняется в базе данных.
Отчёт
По завершении доступна статистика: создано, пропущено (уже есть), ошибок, нет места.

Провижининг обрабатывает пользователей пачками по 10, с задержкой 0.1 секунды между ключами и 0.5 секунды между пачками для снижения нагрузки на 3x-ui.

Балансировка нагрузки

Просмотр баланса

Через меню локации → «Баланс нагрузки» можно посмотреть распределение пользователей между серверами. Показывается:

  • Количество активных ключей на каждом сервере
  • Процент загрузки
  • Средняя загрузка по локации
  • Статус: Сбалансировано (все серверы ±10% от среднего) или Несбалансировано

Ребалансировка

Через меню локации → «Ребалансировка» система автоматически перемещает ключи с перегруженных серверов (>80% или >120% от среднего) на недогруженные (<80% от среднего).

Пример: Локация «Нидерланды» с двумя серверами: NL-1 (800/1000 = 80%) и NL-2 (200/1000 = 20%). После ребалансировки: NL-1 (500/1000 = 50%) и NL-2 (500/1000 = 50%).

Добавление нового сервера

Пошаговый процесс добавления сервера в систему.

Что потребуется

  • VPS с установленной панелью 3x-ui
  • IP-адрес сервера
  • SSH-доступ (root, пароль или ключ)
  • Данные 3x-ui панели (URL, логин, пароль)

Процесс

Создайте локацию (если новая)
Нажмите «Добавить локацию», выберите страну, задайте лимит трафика (если нужен).
Добавьте сервер
В меню локации → «Добавить сервер». Введите IP, SSH-данные, данные 3x-ui панели.
Полная проверка
Нажмите «Полная проверка». Система проверит SSH-соединение, подключение к 3x-ui, запустит speedtest и установит агент. По результатам speedtest рассчитает рекомендуемую вместимость.
Сохраните
После успешной проверки нажмите «Сохранить». Система автоматически начнёт провижининг ключей для всех пользователей с подпиской.
Важно

Система автоматически проверяет уникальность IP — нельзя добавить два сервера с одним и тем же адресом. При вводе IP проверка происходит мгновенно.

Расчёт вместимости

Рекомендуемое количество пользователей рассчитывается по формуле на основе speedtest:

рекомендуемых пользователей = (скорость_mbps × 0.75) / (8 mbps × 0.15)

Где 8 Mbps — средняя скорость на одного пользователя, 0.15 — коэффициент одновременных подключений, 0.75 — запас по каналу.

Пример: Speedtest показал 500 Mbps. Рекомендуемая вместимость: (500 × 0.75) / (8 × 0.15) = 312 пользователей.
