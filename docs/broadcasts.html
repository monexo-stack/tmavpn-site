Рассылки

Массовая отправка сообщений пользователям через Telegram-бота. Поддержка фильтров аудитории, изображений, отложенной отправки и прогресса в реальном времени.

Создание рассылки

Рассылка создаётся в разделе Broadcasts админ-панели через модальное окно.

Параметры рассылки

ПараметрОписание
АудиторияФильтр получателей (подробнее ниже)
Текст сообщенияПоддержка HTML-тегов: <b>, <i>, <u>, <code>
ИзображениеJPG, PNG, GIF, WEBP до 10 МБ. Загружается через drag-and-drop или выбор файла
Время отправкиНемедленно или по расписанию (datetime picker)

Перед отправкой система показывает предварительный подсчёт получателей.

Фильтры аудитории

ФильтрКого выберет
Все пользователи Все зарегистрированные (исключая забаненных и заблокировавших бота)
С активной подпиской Пользователи с оплаченной подпиской (subscription_expires_at > сейчас)
Без подписки Пользователи без активной подписки
Подписка истекает Подписка заканчивается в ближайшие 7 дней (исключая trial)
Конкретный пользователь Один пользователь по Telegram ID (с поиском)

Статусы рассылки

СтатусОписание
draftЧерновик, ещё не отправлялась
scheduledЗапланирована на определённое время
sendingИдёт отправка сообщений
completedОтправка завершена
failedОшибка при отправке

Процесс отправки

Рассылка использует высокопроизводительный механизм с адаптивным rate limiting.

Технические характеристики

ПараметрЗначение
Параллельные воркеры5 одновременных потоков отправки
Скорость20–28 сообщений/сек (адаптивная)
Задержка между сообщениями35–1000 мс (авто-увеличивается при rate limit)
Размер батча1000 пользователей
Повторные попыткиДо 2 на каждого пользователя с паузой 5 сек
Обновление прогрессаКаждые 100 сообщений

Прогресс отправки

Во время отправки админ-панель показывает:

  • Прогресс — процент завершения (sent + failed + blocked) / total
  • Скорость — сообщений в секунду
  • ETA — оставшееся время (в минутах)
  • Sent — успешно доставлено
  • Failed — ошибки доставки
  • Blocked — пользователь заблокировал бота
Возобновление отправки

Если рассылка была прервана (перезагрузка сервера, ошибка), её можно возобновить. Система пропустит пользователей, которым уже было отправлено сообщение (проверка по broadcast_logs).

Управление рассылками

  • Отправить — запустить отправку черновика или запланированной рассылки
  • Остановить — приостановить текущую рассылку
  • Возобновить — продолжить прерванную рассылку
  • Удалить — удалить черновик или запланированную рассылку
  • Логи — посмотреть детальные логи доставки для каждого пользователя
