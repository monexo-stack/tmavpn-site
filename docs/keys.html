Ключи и подписки

VPN-ключи создаются автоматически при покупке подписки. Один пользователь получает отдельный ключ на каждую локацию, объединённый единым токеном подписки.

Как устроены ключи

Каждый ключ — это конфигурация VPN (VLESS + Reality), созданная на конкретном сервере. При покупке подписки система создаёт ключи на всех доступных локациях и объединяет их одним токеном подписки.

Один токен — все локации
Пользователь получает одну ссылку подписки, которая содержит конфигурации всех серверов. Токен генерируется один раз и не меняется.
Автоматический провижининг
При добавлении новой локации ключи автоматически создаются для всех активных подписчиков. При перевыпуске — для недостающих локаций.
Лимит устройств
Ограничение одновременных подключений определяется тарифом пользователя. Лимит устройств передаётся на VPN-сервер автоматически.
Отслеживание трафика
VPN Agent на каждом сервере передаёт данные о трафике через WebSocket в реальном времени. Суммарный трафик хранится в профиле пользователя.

Связи между сущностями

СвязьТипОписание
Пользователь → Ключи1:NУ пользователя может быть несколько ключей (по одному на каждую локацию)
Ключ → СерверN:1Каждый ключ создан на конкретном сервере
Сервер → ЛокацияN:1Сервер принадлежит локации (стране)
Токен подпискиГруппировкаВсе ключи одного пользователя объединены одним токеном
Пример

Пользователь с тарифом «1 год» и 3 локациями (Германия, Нидерланды, США) получит 3 ключа с одним токеном подписки. Все три ключа доступны по одной ссылке подписки.

Фильтры и поиск

  • Глобальный поиск — по имени пользователя, ID, токену подписки
  • Локация — фильтр по конкретной локации (выпадающий список)
  • Статус — active или expired
  • Bot Score — показать пользователей с высоким риском (боты)

Сортировка доступна по колонкам: ID, дата создания, дата истечения, пользователь.

Статусы ключей

Статус ключа определяется двумя факторами: включён ли ключ и не истекла ли подписка пользователя.

СтатусУсловиеОписание
Active Ключ включён И подписка не истекла Ключ работает, пользователь может подключаться к VPN
Expired Подписка истекла ИЛИ ключ отключён Подписка истекла или ключ отключён вручную
Важно

Ключ считается активным только если оба условия выполнены: ключ включён и дата подписки ещё не истекла. Если хотя бы одно из условий не выполнено — ключ отображается как expired.

Действия с ключами

Действия в контекстном меню

Для каждого ключа в таблице доступно выпадающее меню с действиями:

ДействиеЧто делаетКогда использовать
View Details Открывает детальную карточку ключа: все локации, история устройств, тариф Для диагностики проблем с подключением пользователя
Reissue Key Удаляет старый ключ на сервере, создаёт новый. Автоматически создаёт ключи для недостающих локаций Если ключ не работает, если добавлены новые локации, если нужна новая конфигурация
Extend Subscription Добавляет дни к дате истечения подписки (по умолчанию +7 дней) Компенсация за проблемы, подарок пользователю
Delete Удаляет ключ и с VPN-сервера. Удаляются все ключи под одним токеном подписки Полная отмена подписки, удаление тестовых ключей

Массовые действия

При выборе нескольких ключей через чекбоксы доступны:

  • Bulk Reissue — перевыпуск выбранных ключей (с задержкой 500мс между запросами для снижения нагрузки)
  • Bulk Delete — удаление выбранных ключей (с задержкой 300мс между запросами)
  • Hide Inactive — скрытие expired ключей из списка (клиентская фильтрация)

Детальная карточка ключа

При нажатии View Details открывается модальное окно с полной информацией:

  • Локации — список всех серверов, на которых есть ключ пользователя, с IP-адресами
  • История устройств — какие устройства подключались, когда и откуда
  • Тарифный план — текущий план, лимит устройств, дата истечения
  • Количество перевыпущенных и автоматически созданных ключей — после операции reissue

Токен подписки

Subscription token — это уникальный идентификатор, связывающий все ключи пользователя в одну подписку.

Характеристики токена

ПараметрЗначение
Генерация8 символов, уникальный
УникальностьГарантируется системой при генерации
ПривязкаХранится в профиле пользователя и в каждом ключе
ПостоянствоСоздаётся один раз при первой покупке. Не меняется при продлении подписки
Формат URLhttps://domain/sub/{token}

Как работает ссылка подписки

Пользователь копирует URL
Ссылка имеет формат https://vpn.example.com/sub/aBc123dE. Один и тот же URL используется на протяжении всего времени использования сервиса.
VPN-клиент запрашивает конфигурацию
Клиент (v2RayTun, Hiddify, v2rayNG) делает GET-запрос на URL подписки. Сервер находит все ключи по токену.
Сервер возвращает агрегированную конфигурацию
Ответ содержит конфигурации всех активных серверов пользователя. VPN-клиент получает список всех доступных локаций.
Пользователь выбирает сервер и подключается
В списке серверов VPN-клиента появляются все локации. Пользователь выбирает нужную и нажимает «Подключить».

Жизненный цикл ключа

Покупка подписки
Пользователь оплачивает тариф. Система генерирует токен подписки (если первая покупка), определяет лимит устройств из тарифа и начинает провижининг ключей.
Провижининг на серверы
Для каждой активной локации система выбирает лучший сервер (по загрузке), создаёт клиент на VPN-панели, получает VLESS-конфигурацию и сохраняет ключ.
Активное использование
Ключ активен, пользователь подключается через VPN-клиент. VPN Agent отправляет данные о трафике через WebSocket. Система отслеживает устройства и подключения.
Истечение подписки
Дата окончания подписки проходит. Фоновая задача проверяет истёкшие подписки. Ключи отключаются, пользователь получает уведомление через бота.
Продление или удаление
При продлении: дата подписки обновляется, ключи включаются обратно, трафик сбрасывается. При удалении: ключи удаляются с VPN-серверов, счётчик пользователей на серверах обновляется.

Перевыпуск ключей (Reissue)

Операция reissue выполняет полное пересоздание ключей пользователя:

  1. Находит все ключи пользователя по токену подписки
  2. Для каждого существующего ключа: удаляет старый ключ на сервере, создаёт новый
  3. Проверяет, есть ли локации без ключей (новые локации, добавленные после покупки)
  4. Для недостающих локаций: выбирает сервер, создаёт клиент, сохраняет ключ
Результат перевыпуска

Ответ содержит: количество перевыпущенных ключей, количество новых ключей (для новых локаций), общее количество и список ошибок (если были).

Каскадное удаление

При удалении ключа удаляются все ключи под тем же токеном подписки. Это сделано по дизайну — подписка работает как единое целое. Дополнительно у пользователя сбрасываются:

  • Дата окончания подписки
  • Текущий тарифный план
  • Лимит устройств
  • Данные уведомлений

При этом токен подписки, дата первой подписки и суммарный трафик сохраняются для истории.

Трафик

Как собирается трафик

Данные о трафике приходят в реальном времени от VPN Agent, установленного на каждом сервере.

Agent отправляет данные
VPN Agent на сервере собирает статистику и отправляет данные о трафике каждого клиента через WebSocket.
Идентификация пользователя
Система автоматически определяет, какому пользователю принадлежит трафик на каждом сервере.
Суммирование по серверам
Трафик суммируется по всем серверам пользователя и сохраняется в профиле.

Лимиты трафика

Тип лимитаГде задаётсяКак работает
Трафик на локацию Настройка локации Лимит трафика задаётся для каждой локации. Если не задан — безлимитно
Устройства Тарифный план Ограничение одновременных подключений. Лимит устройств передаётся на VPN-сервер

Сброс трафика

  • Ежемесячный сброс — фоновая задача сбрасывает счётчики трафика для каждой локации. Дата последнего сброса фиксируется в ключе
  • При продлении подписки — трафик сбрасывается на сервере автоматически

Домен подписки

По умолчанию ссылки подписки используют основной домен платформы. Можно настроить отдельный домен для ссылок подписки.

Настройка домена

Укажите домен
В настройках админ-панели введите домен (например, sub.vpn.example.com). Домен должен быть направлен на IP сервера через A-запись в DNS.
Проверка DNS
Система проверяет, что домен указывает на правильный IP. Если DNS не настроен — выдаст ошибку.
Настройка Nginx и SSL
Автоматически настраивается конфигурация Nginx на хосте и получается SSL-сертификат через Let's Encrypt. Процесс занимает 1–3 минуты.
Активация
После успешной настройки все новые ссылки подписки будут использовать новый домен. Существующие ссылки обновляются автоматически.
Зачем отдельный домен

Если основной домен заблокируют, ссылки подписки перестанут работать. Отдельный домен для подписок позволяет быстро переключиться на новый адрес без потери доступа пользователей к VPN.
