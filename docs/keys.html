Ключи и подписки

VPN-ключи создаются автоматически при покупке подписки. Один пользователь получает отдельный ключ на каждую локацию, объединённый единым токеном подписки.

Как устроены ключи

Каждый ключ — это конфигурация VPN (VLESS + Reality), созданная на конкретном сервере через 3x-ui панель. При покупке подписки система создаёт ключи на всех доступных локациях и объединяет их одним subscription_token.

Один токен — все локации
Пользователь получает одну ссылку подписки, которая содержит конфигурации всех серверов. Токен генерируется один раз и не меняется.
Автоматический провижининг
При добавлении новой локации ключи автоматически создаются для всех активных подписчиков. При перевыпуске — для недостающих локаций.
Лимит устройств
Ограничение одновременных подключений задаётся через параметр limit_ip в 3x-ui. Определяется тарифом пользователя.
Отслеживание трафика
VPN Agent на каждом сервере передаёт данные о трафике через WebSocket в реальном времени. Суммарный трафик хранится в профиле пользователя.

Связи между сущностями

СвязьТипОписание
Пользователь → Ключи1:NУ пользователя может быть несколько ключей (по одному на каждую локацию)
Ключ → СерверN:1Каждый ключ создан на конкретном сервере
Сервер → ЛокацияN:1Сервер принадлежит локации (стране)
subscription_tokenГруппировкаВсе ключи одного пользователя объединены одним токеном
Пример

Пользователь с тарифом «1 год» и 3 локациями (Германия, Нидерланды, США) получит 3 ключа с одним subscription_token. Все три ключа доступны по одной ссылке подписки.

Статусы ключей

Статус ключа определяется двумя факторами: флагом active в базе данных и датой истечения подписки пользователя.

СтатусУсловиеОписание
Active key.active = true И subscription_expires_at > now Ключ работает, пользователь может подключаться к VPN
Expired subscription_expires_at ≤ now ИЛИ key.active = false Подписка истекла или ключ отключён вручную
Важно

Ключ считается активным только если оба условия выполнены: флаг active = true в базе данных и дата подписки ещё не истекла. Если хотя бы одно из условий не выполнено — ключ отображается как expired.

Массовые действия

При выборе нескольких ключей через чекбоксы доступны:

  • Bulk Reissue — перевыпуск выбранных ключей (с задержкой 500мс между запросами для снижения нагрузки)
  • Bulk Delete — удаление выбранных ключей (с задержкой 300мс между запросами)
  • Hide Inactive — скрытие expired ключей из списка (клиентская фильтрация)

Детальная карточка ключа

При нажатии View Details открывается модальное окно с полной информацией:

  • Локации — список всех серверов, на которых есть ключ пользователя, с IP-адресами
  • История устройств — какие устройства подключались, когда и откуда
  • Тарифный план — текущий план, лимит устройств, дата истечения
  • Количество перевыпущенных и автоматически созданных ключей — после операции reissue

Токен подписки

Subscription token — это уникальный идентификатор, связывающий все ключи пользователя в одну подписку.

Характеристики токена

ПараметрЗначение
Генерацияsecrets.token_urlsafe(8) — 8 символов, URL-safe
УникальностьПроверяется в базе данных при генерации (до 10 попыток)
ПривязкаХранится в Persons.subscription_token и в каждом Keys.subscription_token
ПостоянствоСоздаётся один раз при первой покупке. Не меняется при продлении подписки
Формат URLhttps://domain/sub/{token}

Как работает ссылка подписки

Пользователь копирует URL
Ссылка имеет формат https://vpn.example.com/sub/aBc123dE. Один и тот же URL используется на протяжении всего времени использования сервиса.
VPN-клиент запрашивает конфигурацию
Клиент (v2RayTun, Hiddify, v2rayNG) делает GET-запрос на URL подписки. Сервер находит все ключи по токену.
Сервер возвращает агрегированную конфигурацию
Ответ содержит конфигурации всех активных серверов пользователя. VPN-клиент получает список всех доступных локаций.
Пользователь выбирает сервер и подключается
В списке серверов VPN-клиента появляются все локации. Пользователь выбирает нужную и нажимает «Подключить».
Именование клиентов на 3x-ui

На каждой 3x-ui панели клиент создаётся с именем {telegram_id}_vpn. Например, для пользователя с Telegram ID 381799724 имя будет 381799724_vpn. Это же имя используется для поиска, удаления и управления клиентом на панели.

Жизненный цикл ключа

Покупка подписки
Пользователь оплачивает тариф. Система генерирует subscription_token (если первая покупка), определяет лимит устройств из тарифа и начинает провижининг ключей.
Провижининг на серверы
Для каждой активной локации система выбирает лучший сервер (по загрузке), создаёт клиент на 3x-ui панели через ServerManager.create_client(), получает VLESS-конфигурацию и сохраняет ключ в базу данных.
Активное использование
Ключ активен, пользователь подключается через VPN-клиент. VPN Agent отправляет данные о трафике через WebSocket. Система отслеживает устройства и подключения.
Истечение подписки
Дата subscription_expires_at проходит. Фоновая задача проверяет истёкшие подписки каждые 60 секунд. Ключи отключаются, пользователь получает уведомление через бота.
Продление или удаление
При продлении: дата подписки обновляется, ключи включаются обратно, трафик сбрасывается. При удалении: клиенты удаляются с 3x-ui, записи удаляются из базы, счётчик current_users на серверах уменьшается.

Перевыпуск ключей (Reissue)

Операция reissue выполняет полное пересоздание ключей пользователя:

  1. Находит все ключи пользователя по subscription_token
  2. Для каждого существующего ключа: удаляет старый клиент на 3x-ui, создаёт новый с новым UUID
  3. Проверяет, есть ли локации без ключей (новые локации, добавленные после покупки)
  4. Для недостающих локаций: выбирает сервер, создаёт клиент, сохраняет ключ
Результат перевыпуска

Ответ содержит: reissued_count (перевыпущенные), provisioned_count (новые локации), total (общее количество), errors (ошибки, если были).

Каскадное удаление

При удалении ключа удаляются все ключи под тем же subscription_token. Это сделано по дизайну — подписка работает как единое целое. Дополнительно очищаются поля пользователя:

  • subscription_expires_at → null
  • current_plan_id → null
  • device_limit → 0
  • last_notification_key, last_notification_at → null

При этом subscription_token, subscription_created_at и total_traffic сохраняются для истории.

Трафик

Как собирается трафик

Данные о трафике приходят в реальном времени от VPN Agent, установленного на каждом сервере.

Agent отправляет данные
VPN Agent на сервере собирает статистику с 3x-ui панели и отправляет список клиентов с up_bytes и down_bytes через WebSocket.
Идентификация пользователя
Telegram ID извлекается из имени клиента: 381799724_vpn → 381799724.
Суммирование по серверам
Трафик суммируется по всем серверам пользователя. Результат записывается в Persons.total_traffic (в байтах).

Лимиты трафика

Тип лимитаГде задаётсяКак работает
Трафик на локацию Location.traffic_limit_gb Устанавливается через limit_gb при создании клиента на 3x-ui. Null или 0 — безлимитно
Устройства Persons.device_limit (из тарифа) Ограничение одновременных подключений через limit_ip на 3x-ui

Сброс трафика

  • Ежемесячный сброс — фоновая задача сбрасывает счётчики трафика на 3x-ui для каждой локации. В ключе обновляется traffic_reset_at
  • При продлении подписки — трафик сбрасывается автоматически через reset_client_traffic() на 3x-ui

Домен подписки

По умолчанию ссылки подписки используют основной домен платформы. Можно настроить отдельный домен для ссылок подписки.

Настройка домена

Укажите домен
В настройках админ-панели введите домен (например, sub.vpn.example.com). Домен должен быть направлен на IP сервера через A-запись в DNS.
Проверка DNS
Система проверяет, что домен указывает на правильный IP. Если DNS не настроен — выдаст ошибку.
Настройка Nginx и SSL
Автоматически настраивается конфигурация Nginx на хосте и получается SSL-сертификат через Let's Encrypt. Процесс занимает 1–3 минуты.
Активация
После успешной настройки все новые ссылки подписки будут использовать новый домен. Существующие ссылки в базе данных обновляются автоматически.
Зачем отдельный домен

Если основной домен заблокируют, ссылки подписки перестанут работать. Отдельный домен для подписок позволяет быстро переключиться на новый адрес без потери доступа пользователей к VPN.
